# Urban Routes – Web Application Testing Project

This project focuses on manual web application testing practices applied to the Urban Routes platform.

The project demonstrates UI validation, layout testing, cross-browser testing, checklist execution, functional validation, and bug reporting using structured QA documentation techniques.

---

# Project Scope

The project was designed to validate the behavior, layout, and functionality of the Urban Routes web application across different browsers and interface scenarios.

Testing activities included:

- Functional Testing
- UI Testing
- Layout Validation
- Cross-browser Testing
- Checklist-based Testing
- Bug Reporting
- User Interface Validation

---

# QA Activities Performed

- Executed structured manual testing scenarios
- Validated web interface layout and responsiveness
- Performed cross-browser testing
- Verified application behavior in different resolutions
- Executed checklist-based validations
- Identified and documented UI defects
- Linked bug reports using JIRA
- Validated expected versus actual results
- Performed regression validation scenarios
